IPV6下利用Cloudflare API设置群辉DDNS实现公网访问
2021-02-25|日常折腾
字数总计: 1.2k|阅读时长: 5 分钟

  目前我国大部分运营商要个ipv4公网IP难得要死，并且提供的宽带服务叠加了多层NAT，进行内网穿透的效果并不理想，但是随着IPV6建设的推进，大部分地区都是可以获取到IPV6地址的，而且IPV6不会被NAT，即便是路由器下的多台设备，每台获取到的都直接是公网地址，我们如果把NAS的IPV6地址进行DDNS设置，那么实际上在支持IPV6的环境下就可以直接访问了，而且不必再进行端口转发之类的。

  群辉的DDNS功能并不支持Cloudflare，所以本文借助Cloudflare API通过编写定时脚本的方式来实现自动更新IPV6地址。

Cloudflare设置

新建解析记录

  建立解析记录之前首先要把域名解析到Cloudflare，这个就不说了。

  在DNS里面新建一条AAAA记录，并关掉CDN功能，即后面代理状态改为仅限DNS。A记录是IPV4的，AAAA是IPV6的，具体地址可以随便填，因为后续会自动更新。例如我这里建了一条叫做ipv6的AAAA记录。

申请API Key

  在右上角点击个人资料，然后在API令牌页面里面获取Global API Key，由于我这里已经获取过，可能按钮不太一样。
